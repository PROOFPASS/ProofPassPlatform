## How to Become a Contributor

We welcome contributions from the community. To get started:

1. Read our [Contributing Guidelines](CONTRIBUTING.md)
2. Check the [Code of Conduct](CODE_OF_CONDUCT.md)
3. Look for issues labeled `good-first-issue` or `help-wanted`
4. Submit a pull request

## Recognition

Contributors who submit merged pull requests will be:
- Listed in this file
- Mentioned in release notes
- Acknowledged in the project README

## Types of Contributions

We recognize various forms of contribution:

- **Code**: New features, bug fixes, refactoring
- **Documentation**: Improving docs, adding examples, translations
- **Testing**: Writing tests, reporting bugs, QA
- **Design**: UI/UX improvements, graphics, branding
- **Community**: Answering questions, organizing events, advocacy
